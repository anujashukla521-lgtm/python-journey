###  Python OOP - Inheritance
## Overview
This folder contains programs created while learning Inheritance, Access Specifiers, Encapsulation, and Constructor Inheritance in Python. The programs demonstrate how child classes can inherit attributes and methods from parent classes, how access levels are represented in Python, and how encapsulation helps protect object data.

## Topics Covered
- Single Inheritance
- Constructor Inheritance
- super() Function
- Public Members
- Protected Members (_variable)
- Private Members (__variable)
- Name Mangling
- Encapsulation
- Getters and Setters
- @property Decorator
- Data Validation
  
## Key Learnings
- Inheritance promotes code reusability.
- Child classes can access parent class methods and attributes.
- The super() function allows a child class to call the parent constructor.
- Public, protected, and private members in Python are implemented through naming conventions and name mangling.
- Encapsulation helps protect sensitive data from direct modification.
- Properties and setters provide a clean way to validate data before storing it.
